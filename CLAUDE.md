## Build & Apply Commands

```bash
# Rebuild the full system (also available as alias: nixrebuild)
sudo nixos-rebuild switch --flake ~/nixos-config#nixos && sudo systemctl daemon-reload

# Rebuild home-manager config only (also available as alias: homerun)
home-manager switch --flake .

# Run Neovim from flake
nix run ~/neovim-flake#

# Run LazyVim variant
NVIM_APPNAME=lazyvim nvim
```

There are no tests or linters — this is a declarative NixOS config, not a software project. Validation happens at build time via `nixos-rebuild`.

## Architecture

This is a **flake-based NixOS configuration** for a single machine (`nixos`). Home-manager is integrated at the system level (not standalone), meaning `nixos-rebuild switch` applies both system and user config in one shot.

### Entry Points

- **`flake.nix`** — defines all inputs and the single `nixosConfigurations.nixos` output; passes `inputs` to all modules
- **`configuration.nix`** — system packages, boot, networking, users, display manager; imports from `system/`
- **`home.nix`** — user shell (Zsh + oh-my-zsh), Neovim (LazyVim), home packages, shell aliases; imports from `user/`
- **`hardware-configuration.nix`** — auto-generated, do not edit

### Module Split: `system/` vs `user/`

System-level NixOS modules live in `system/`; home-manager user modules live in `user/`. Several features (notably Hyprland) are split across both:

- `system/WM/hyprland.nix` — enables the WM and installs system packages
- `user/WM/hyprland.nix` — keybindings, animations, window rules (home-manager)
- `hypr/` — raw Hyprland config files (legacy, not managed by home-manager)

### Key Flake Inputs

| Input | Purpose |
|---|---|
| `nixos-unstable` | NixOS packages channel |
| `home-manager` | User environment management |
| `lazyvim-nix` | LazyVim Neovim configuration framework |
| `quickshell` | Qt-based shell components |
| `dms-plugin-registry` | Plugins for DMS-Shell (`system/WM/dms.nix`) |

### Conventions

- All modules receive the `inputs` argument to access flake inputs (e.g., for quickshell, dms-plugin-registry packages).
- New system-level features belong in `system/`; user-facing program config belongs in `user/`.
- `hardware-configuration.nix` is machine-specific and auto-generated by `nixos-generate-config`.
